# kotlin-serialization

## Objects -> Primitives -> Output format

1. Serializer가 객체를 원시타입의 직렬 시퀀스로 변환합니다.
2. 변환된 원시 값 시퀀스를 Encoder에 의해 출력 형식으로 변환됩니다.

여기에서 인코딩은 데이터를 특정 형식으로 변환하는 전체 과정을 의미합니다.  
kotlin-serialization은 인코딩에 encodeToString(value) 함수를 사용할 수 있고, 해당 value는 @Serializable 어노테이션을 명시해야만 합니다.

- 이유? kotlin-serialization은 직렬화에 리플렉션을 사용하지 않아 직렬화할 수 없는 클래스를 역직렬화하지 않기 위해 명시적으로 어노테이션을 선언하도록 했습니다.

반대로 역직렬화를 하기 위해서는 decodeFromString 함수를 사용할 수 있습니다.  
해당 함수는 타입을 명시적으로 작성하여 매개변수로는 역직렬화하기 위한 데이터를 담아 원하는 객체형태로 반환받을 수 있습니다.

## Serializable 어노테이션

다음과 같은 특징이 있습니다.

- backing fields 직렬화
    - 클래스 프로퍼티 중, backing fields가 있는 것만 직렬화됩니다.
    - setter/getter가 있지만 backing field가 없거나 위임된 속성은 직렬화되지 않습니다.
        - 복습
            - backing field
                - Kotlin 컴파일러가 자동 생성하는 실제 값 저장공간
                    - 자동으로 생성되는 경우
                        - 기본 getter/setter 사용
                        - custom setter에서 field 사용
                        - custom getter에서 field 사용
                        - 초기화값 존재
            - backing property
                - 개발자가 직접 만든 프로퍼티로, 실제 저장 프로퍼티를 외부에 노출시키지 않기 위해 사용
              ```kotlin
                private var _name: String = "myName"
                var name: String
                    get() = _name
                    set(value) {
                        _name = value.ifEmpty { name }       
                    } 
              ```
- 생성자 프로퍼티 필수
    - 클래스의 기본 생성자의 모든 매개변수는 속성이어야만 합니다.
    ```kotlin
        // as-is
        @Serializable
        class Project(path: String) {
            val owner: String = path.substringBefore('/')
            val name: String = path.substringAfter('/')
        } 
  
        // to-be
        @Serializable
        class Project private constructor(val owner: String, val name: String) {
            constructor(path: String): this(
                owner = path.substringBefore('/'),
                name = path.substringAfter('/')
            )
            
            val path: String
                get() = "$owner/$name"
        }     
    ```
- 데이터 유효성 검사
    - 값을 검증하고 나서 프로퍼티를 저장하고 싶을 때, 직렬화 하기 위해서는 생성자 변수를 init 블록에서 검증해야 합니다.
- Optional 프로퍼티
    - 객체는 모든 속성이 입력에 존재할 때만 역직렬화 될 수 있습니다.
    - 만약에 역직렬화하려는 데이터가 객체의 모든 프로퍼티를 만족시키지 못한다면 MissingFieldException이 발생합니다.
    - 필수인 프로퍼티가 아닌 경우 기본값을 할당한다면 예외를 해결할 수 있습니다.
- Optional 프로퍼티 초기화 호출
    - Optional 프로퍼티에 기본 값으로 초기화 함수를 설정하더라도 값이 지정된다면 해당 함수는 호출되지 않습니다.
- 필수 프로퍼티
    - 만약에 기본값이 있는 프로퍼티여도 `@Required` 어노테이션이 프로퍼티에 선언되어 있을 경우, 해당 프로퍼티를 채우지 않으면 예외를 발생시킬 수 있습니다.
- Transient 프로퍼티
    - @Transient 어노테이션을 표시해서 직렬화에서 제외할 수 있습니다.
    - @Transient 어노테이션이 표시된 프로퍼티는 반드시 기본값이 존재해야 합니다.
    - 만약 @Transient 어노테이션이 표시된 프로퍼티를 역직렬화하려고 선언된 경우 예외를 발생시킵니다.
- 기본값은 인코딩되지 않습니다.
    - default value는 JSON에서 인코딩 되지 않는 것이 기본값입니다.
    - @EncodeDefault를 사용하면 속성의 값이나 형식 설정에 관계없이 항상 직렬화하도록 할 수 있습니다.
        - EncodeDefault의 Mode를 사용해서 인코딩을 하는 경우를 지정할 수 있습니다.
            - @EncodeDefault(EncodeDefault.Mode.ALWAYS): serializer를 통해 기본값과 같더라도 항상 인코딩 하도록 합니다. (Default)
            - @EncodeDefault(EncodeDefault.Mode.NEVER): serializer를 통해 기본값과 같을경우 인코딩 하지 않도록 합니다.
- Nullable 프로퍼티
    - nullable 프로퍼티를 지원합니다.
- 타입 안전성 강제
    - null 값을 허용하지 않는 프로퍼티에 null 값을 담으려고 한다면 예외를 발생시킵니다.
- Referenced objects
    - 직렬화 가능한 클래스는 내부 프로퍼티로 직렬화 가능한 다른 클래스를 참조할 수 있습니다.
- No compression of repeated references
    - kotlin-serialization은 일반 데이터의 인코딩, 디코딩을 위해 설계되었습니다. 반복된 객체 참조가 있는 객체 그래프의 재구성을 지원하지 않습니다.
    - 객체 참조를 보존하지 않고, 동일한 인스턴스를 2개의 프로퍼티에 할당하더라도 동일한 참조가 아닌 각각 독립적인 값으로 직렬화합니다.
    - 또한, 순환 구조를 직렬화하려고 시도할 경우 stack overflow가 발생합니다.
- Generic classes
    - 제네릭 클래스는 kotlin serialization이 컴파일 타임에 강제되기에 다형성을 활용할 수 있습니다.
    - 하지만 다형적 타입인 T 타입은 직렬화 가능한 타입이어야만 합니다.
- Serial field names
    - 인코딩된 이름은 기본적으로 프로퍼티의 이름과 동일하게 판단합니다.
    - 프로퍼티 이름을 다른 이름으로 인코딩하려면 @SerialName으로 지정할 수 있습니다.

## builtin-classes

기본 원시타입 + String + 열거형 + 복합체(Pair, Triple) + Collections(List, Set, Map) + Unit + Singleton + Duration + Nothing 타입들을
직렬화할 수 있습니다.

- 문자열로 된 긴 숫자의 경우
    - Kotlin Long의 전체 범위는 JavaScript 숫자에 맞기 않기 때문에, 정밀도가 JavaScript에서 손실됩니다.
    - 이런 경우 문자열로 완전한 긴 숫자를 나타낼 수 있습니다.
        - Kotlin Serialization에서는 LongAsStringSerializer 를 지원합니다.
          ```kotlin
             @Serializable(with=LongAsStringSerializer::class)
             val signature: Long
          ```
- 열거형 클래스
    - 모든 열거형 클래스는 @Serializable 어노테이션을 표시하지 않고도 바로 직렬화할 수 있습니다.
    - Kotlin/JS 및 Kotlin/Native 에서는 열거형 클래스를 root object로 사용하려면 @Serializable 어노테이션이 필요합니다.
    - 열거형 상수에 @SerialName을 설정해주기 위한다면 @Serializable 어노테이션이 필요합니다.
- Singleton 객체
    - 싱글턴은 오직 하나의 인스턴스만 있는 클래스로, 객체 자체가 자신의 상태를 정의합니다.
    - JSON에서는 Unit과 동일하게 객체가 빈 구조로 직렬화 됩니다.
- Nothing
    - 직렬화 가능한 클래스이지만, 클래스의 인스턴스가 없기에 인코딩 그리고 디코딩하는 것이 불가능하며, 예외를 발생시킵니다.

## Serializer

- @Serializable 어노테이션이 표시된 클래스는 Kotlin Serialization 컴파일러 플러그인에 의해 자동으로 생성된 KSerializer 인터페이스의 인스턴스를 받습니다.
    - 이 인스턴스를 동반 객체에서 .serializer() 함수를 사용해서 가져올 수 있습니다.
    - .serializer() 함수는 해당 클래스의 타입 매개변수 수만큼의 매개변수를 허용합니다.

- Serializer를 직접 구현할 수도 있습니다.
    - KSerializer<T> 을 구현합니다.
        - descriptor 변수
            - encode, decode 함수가 무슨 역할을 하는지 설명하며, 형식 구현이 어떤 인코딩/디코딩 방법을 호출하는 지 미리 알 수 있도록 해야 합니다.
                - PrimitiveSerialDescriptor는 직렬화되는 유형의 고유한 이름과 함께 사용해야 하며, PrimitiveKind는 구현에 사용되는 특정 encode/decode 방법을
                  설명합니다.
        - serialize 함수
            - SerializationStrategy을 구현합니다.
            - Encoder의 인스턴스와 직렬화할 값을 받습니다.
        - deserialize 함수
            - DeserializationStrategy을 구현합니다.
            - Decoder의 인스턴스를 받아서 역직렬화된 값을 반환합니다.
- java.util.Date 타입 같은 직렬화할 수 없는 3rd party 클래스의 경우 직접 Serializer를 구현해서 직렬화, 역직렬화를 수행할 수 있습니다.
    - 해당 프로퍼티에 @Serializable(with = 구현한 Serializer 클래스)
    - 또는 직접 타입에 적용할 수도 있습니다.
       ```kotlin
        @Serializable
        class ProgrammingLanguage(
            val name: String,
            val releaseDates: List<@Serializable(DateAsLongSerializer::class) Date>
        )  
       ```

## polymorphism

- 정적 타입
    - 타입에 대해 완전히 정적입니다. 인코딩된 객체의 구조는 컴파일 타임에 결정됩니다.

```kotlin
@Serializable
open class Project(val name: String)

class OwnedProject(name: String, val owner: String) : Project(name)

fun main() {
    val data = OwnedProject("kotlinx.coroutines", "kotlin")
    println(Json.encodeToString(data))
}
```

1. data 타입을 Project 타입으로 명시할 경우 name에 대해서만 인코딩된 값이 출력됩니다.
2. 현재처럼 명시하지 않은 경우 OwnedProject는 직렬화 가능한 클래스가 아니여서 예외가 발생합니다.
    - OwnedProject에 @Serializable을 붙일 수 없다. -> 생성자 프로퍼티인 name 프로퍼티가 필수여야만 합니다.
    - 그렇다면 Project 클래스가 open이 아닌 abstract면 가능할까?
        1. data 타입을 Project 타입으로 명시하게 된다면 Project 타입이 abstract 여서 실제 구현체를 알 수 없어서 어떤 serializer를 사용해야할지 판단할 수 없어 런타임 에러가
           발생하게 됩니다.
            - 이러한 경우 sealed class 를 사용해서 해결 할 수 있습니다.
        2. OwnedProject 타입인 경우 인코딩된 문자열이 정상 출력됩니다.


- sealed class
    - sealed class의 base 클래스는 백킹 필드가 있는 속성을 가질 수 있습니다.
        - 슈퍼클래스의 속성은 서브클래스의 속성보다 먼저 직렬화됩니다.
    - sealed class는 하위에 객체를 가질 수 있으며, 하위 객체들도 @Serializable로 표시해야 합니다.
        - object는 빈 클래스처럼 직렬화되며, 기본적으로 클래스의 전체 이름(클래스의 위치(패키지명 포함))을 type으로 표시합니다.

```kotlin
@Serializable
sealed class Project {
    abstract val name: String
}

@Serializable
class OwnedProject(override val name: String, val owner: String) : Project()

fun main() {
    val data: Project = OwnedProject("kotlinx.coroutines", "kotlin")
    println(Json.encodeToString(data)) // println(Json.encodeToString<Project>(data))
}  
```

- 출력결과로 type 이라는 키에 구분할 수 있는 값이 추가적으로 들어오게 됩니다.
    - type에 들어오는 기본 값은 완전한 클래스명입니다. @SerialiName 주석으로 이름을 명시 할수 있습니다.
        - SerialName을 사용할 때 서로 다른 클래스에 동일한 이름을 할당하지 않도록 합니다.
- 다만 여기에서 Project 타입을 생략해 OwnedProject으로 변수를 선언하게 될 경우 객체 컴파일 타임이 OwnedProject가 되어 type 키 값이 JSON 형식에 포함되지 않습니다.
- Kotlin Serialization은 직렬화 시 사용된 컴파일 타임 타입과 역직렬화 시 사용된 타입이 동일할 때만 올바르게 작동하도록 설계되었습니다.
