## inline

고차함수를 사용하면 특정 런타임 페널티가 발생합니다.  
함수는 객체이면서, 클로저를 캡처하며 메모리 할당이 필요하고 가상호출은 런타임에 오버헤드를 발생시킵니다.  
이런 상황에서 inline 키워드를 사용하면 컴파일러가 인식할 수 있도록 인라인됩니다.  
인라이닝 된 코드는 메모리 성능을 향상시킬 수 있습니다. 하지만 인라이닝되며 코드의 크기를 증가시킬 수 있으므로, 큰 함수는 인라인하지 않는 것을 권장합니다.
또한 인라인 함수는 아래의 코드 예시와 같이 비지역 반환을 할 수 있습니다.
또한, 커스텀 getter, setter 또는 변수 자체에도 inline 키워드를 사용할 수 있습니다.

```kotlin
fun hasZero(numbers: List<Int>): Boolean {
    numbers.forEach {
        if (it == 0) return true
    }
    return false
}
```

## noinline

inline 키워드를 사용하면 함수 전체가 호출 위치에 인라인됩니다.  
noinline 키워드를 사용한다면 일부 함수 매개변수는 인라인화되는 것을 제한할 수 있습니다.

## crossinline

인라인 함수는 객체로 존재하는 것이 아닌 컴파일 시점에 호출 위치에 소스코드로 인라인되기에 비지역 반환을 할 수 있습니다.  
인라인 함수는 함수 본문이 아닌 다른 실행 컨텍스트에서 호출 할 수 있어, 비지역 반환을 하게 될 경우 의도치 않은 문제를 발생시킬 수 있습니다.  
이러한 경우에서 외부의 제어권을 제한하기 위해 인라인 함수의 람다 매개변수가 비지역 반환을 사용할 수 없도록 crossinline 키워드를 사용할 수 있습니다.

```kotlin
inline fun f(crossinline body: () -> Unit) {
    val f = object : Runnable {
        override fun run() = body()
    }
    // ...
}
```

## reified

매개변수로 전달된 타입에 접근이 필요한 경우에 사용할 수 있습니다.  
inline 함수는 컴파일 시점에 타입에 대한 정보를 알 수 있으므로 reified 키워드와 함께 사용하면 컴파일 시점에 타입에 대한 정보를 제공할 수 있습니다.   
리플렉션으로 타입에 접근해야 하는 문제를 해결 할 수 있습니다. 또한, `is`, `as` 와 같이 언어에서 제공하는 일반 연산자를 할 수 있습니다.

```kotlin
inline fun <reified T> TreeNode.findParentOfType(): T? {
    var p = parent
    while (p != null && p !is T) {
        p = p.parent
    }
    return p as T?
}
```
