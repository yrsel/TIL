## Deep Link

- Deep linking은 인텐트 시스템을 활용해 딥 링크를 앱으로 라우팅하는 표준 Android 플랫폼 기능
    - 인텐트를 통해 사용자 정의 URI부터 표준 웹 스킴, 도메인 등 여러 유형의 딥 링크를 처리하도록 등록할 수 있다.
    - 때에 따라 선택화면으로 앱을 선택해야 한다.
- Android App Links는 신뢰할 수 있는 연결을 구축해 앱과 웹사이트 간의 딥 링크를 검증하는 향상된 딥링크 기능
    - 선택화면 없이 즉시 앱에서 해당 콘텐츠를 확인할 수 있다.

## Deep Link

### 개념

- 인텐트 시스템을 활용하여 딥 링크를 앱으로 라우팅하는 표준 Android 플랫폼 기능
- 웹 브라우저, notification, 광고 등에서 앱으로 진입할 수 있도록 한다.
- 웹사이트 또는 도메인을 참조하는 딥링크의 경우, 원활하고 신뢰할 수 있는 경험을 제공하는 AppLinks 사용을 권장한다.

### 작동원리

- Android의 intents system을 활용해서 딥 링크를 앱으로 라우팅한다.
- Manifest에 수신할 딥 링크 URI와 일치하는 intent filter를 선언해야 한다.

작동 흐름 : 사용자가 링크를 클릭 → Android는 인텐트를 트리거해서 앱으로 라우팅을 시도한다.

Android가 라우팅할 URI를 찾는 순서

1. URI를 처리할 수 있는 기본 앱이 있다면 선택
2. 없다면 URI를 처리할 수 있는 유일한 앱을 선택
3. 여러 앱이 존재한다면 사용자가 선택할 수 있도록 대화상자 제공

- intent filter를 적절하게 설정했더라도 우리 앱으로 진입할 것을 완전히 보장할수는 없다.
    - 사용자에게 제어권을 부여하고 선택권을 제공한다.
- Android12(API 31)부터는, 일반적인 web intent는 해당 web intent에 포함된 특정 도메인에 대해 앱이 승인된 경우에만 앱 내의 활동으로 승인된다. 만약, 앱이 도메인에 대해 승인되지
  않은 경우 web intent는 사용자의 기본 브라우저 앱으로 선정한다.

### 3가지 유형

1. Custom Deep links
    - custom URI scheme으로 앱의 컨텐츠로 이동시키는 딥 링크
    - 개발자가 직접 scheme 및 네비게이션을 제어할 수 있지만, 표준 웹 링크가 아니라는 점과 다른 앱에서 동일한 custom scheme을 사용할 경우 역시 사용자에게 대화상자를 트리거 할 수 있다.

2. Web links
    - 표준 http, https scheme을 사용하는 딥링크
    - 표준 URL이기에 다양하게 사용할 수 있지만, 안드로이드 12 이상에서는 대부분 사용자에게 대화상자를 트리거하며 사용자의 웹 브라우저에서 처리될 가능성이 높으며, 앱으로 라우팅 되지 않을 수 있다.

3. App Links
    - Android6.0(API23) 이후부터 사용할 수 있으며, 검증된 웹 링크
    - website association 과정을 통해, Android 시스템에 도메인 소유권을 증명할 수 있다.
        - 검증이 완료되면, 시스템은 사용자에게 대화상자를 트리거 하지 않고 해당 도메인에 대한 링크를 통해 자동으로 앱으로 바로 라우팅해 신뢰할 수 있고 원활한 경험을 제공한다.

튜립 공유 기능의 흐름

- 사용자 앱을 클릭
    - 앱 설치 O → 앱으로 이동 후 ‘함께 튜립’으로 이동하는 흐름
    - 앱 설치 X → 구글 플레이 스토어로 이동 → 설치 완료 후 실행 시 → 앱으로 이동 후 ‘함께 튜립’으로 이동하는 흐름

→ 사용자의 원활한 경험을 위해 App Links 선택

## App Links

- 검증된 웹사이트 URL이 사용자에게 선택창 없이 바로 앱의 콘텐츠를 열도록 할 수 있다.
    - 가능하게 하려면 Android는 웹사이트에 호스팅된 Digital Asset Links 문서를 통해 웹사이트와 앱 간의 안전하고 검증된 환경을 구축할 수 있다.
        - Digital Asset Links : digital asset links protocol과 api는 다른 앱이나 웹사이트를 검증할 수 있도록 만들어준다. (**`assetlinks.json`**
          파일)
    - App Links는 웹사이트와 앱에서 동일한 콘텐츠에 대해 하나의 URL을 사용하므로, 앱이 설치되지 않은 사용자는 대신 웹사이트로 이동하게 된다.(404 에러를 발생시키지 않는다.)
- AppLink는 도메인 소유권 검증이 필요하므로, 다른 앱이 링크를 가로채는 것을 방지한다.
